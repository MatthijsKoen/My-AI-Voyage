Step 1

Install the Prerequisites

Before anything else, you need Git and VS Code installed on your Windows machine.

A
Install Git for Windows

Download Git from git-scm.com/download/win and run the installer. During installation, use these recommended settings:

Git Installer ‚Äî Recommended Options 
# On the "Choosing the default editor" screen:
‚Üí Select "Use Visual Studio Code as Git's default editor"

# On "Adjusting your PATH environment" screen:
‚Üí Select "Git from the command line and also from 3rd-party software"

# On "Configuring the line ending conversions" screen:
‚Üí Select "Checkout Windows-style, commit Unix-style line endings"

# Leave all other options at their defaults and click through

After installation, verify it worked. Open the Windows Terminal or PowerShell and run:

PowerShell / Terminal 
PS> git --version
# Expected output: git version 2.4x.x.windows.1 (or similar)
If you already have VS Code installed, skip to step B. If not, install VS Code first from code.visualstudio.com ‚Äî it's the free version from Microsoft.
B
Create a GitHub Personal Access Token (PAT)

GitHub no longer accepts your password for Git operations ‚Äî you need a Personal Access Token instead. This is a one-time setup.

GitHub.com ‚Äî Settings
1. Go to github.com ‚Üí click your profile picture ‚Üí Settings
2. Scroll to the bottom of the left sidebar ‚Üí Developer settings
3. Click Personal access tokens ‚Üí Tokens (classic)
4. Click Generate new token (classic)
5. Note: "My AI Voyage blog access"
6. Expiration: No expiration (or 1 year)
7. Scopes: check ‚úì repo (full repository access)
8. Click Generate token ‚Üí COPY AND SAVE IT NOW ‚Äî you won't see it again!
Save your token in a secure place (like a password manager). Treat it like a password ‚Äî never put it in your blog files or share it publicly.
Step 2

Configure Git on Your Machine

Tell Git who you are ‚Äî this information appears in your commit history on GitHub.

1
Set Your Identity

Open PowerShell or the Windows Terminal and run these two commands, substituting your own name and the email address linked to your GitHub account:

PowerShell ‚Äî Run Once
PS> git config --global user.name "Matthijs Koen"
PS> git config --global user.email "[email protected]"

# Verify the settings are saved correctly:
PS> git config --global --list
The --global flag means this applies to all your Git projects on this machine, not just this one. You only need to do this once per computer.
2
Store Your GitHub Credentials

Tell Git to remember your credentials so you don't have to enter your token every time you push:

PowerShell ‚Äî Run Once
PS> git config --global credential.helper manager

# This uses Windows Credential Manager to securely store your token.
# The first time you push, Windows will prompt you for credentials ‚Äî
# enter your GitHub username and paste your PAT as the password.
Step 3

Connect Your Local Folder to GitHub

Your index.html already exists locally. Now we link that folder to your GitHub repository.

1
Navigate to Your Project Folder

Open PowerShell and navigate to your project directory:

PowerShell
PS> cd "C:\MyFiles\MattieAI\my-ai-voyage-site"

# Confirm you're in the right place:
PS> dir
# You should see: index.html listed here
2
Initialize Git in This Folder

Turn your project folder into a Git repository:

PowerShell
PS> git init

# Expected output:
# Initialized empty Git repository in C:/MyFiles/MattieAI/my-ai-voyage-site/.git/
3
Create a .gitignore File

A .gitignore file tells Git which files to not track. Create this now to keep your repository clean:

PowerShell ‚Äî Create .gitignore
PS> New-Item -Name ".gitignore" -ItemType "file"
PS> notepad .gitignore

In Notepad, paste the following content and save:

.gitignore ‚Äî Contents
# Windows system files
Thumbs.db
desktop.ini
.DS_Store

# VS Code settings (optional ‚Äî remove this line if you
# want to share your VS Code settings with collaborators)
.vscode/

# API keys and secrets ‚Äî NEVER commit these
.env
secrets.txt
api_keys.txt

# Node modules if you ever add JavaScript dependencies
node_modules/

# Temporary files
*.tmp
*.log
4
Make Your First Commit

Stage all your files and create your very first commit ‚Äî this is the foundation of your repository's history:

PowerShell
# Stage ALL files in the folder for tracking
PS> git add .

# Check what's been staged (optional but recommended)
PS> git status

# Create your first commit with a message
PS> git commit -m "üöÄ Initial commit ‚Äî My AI Voyage blog launch"

# Rename the default branch to 'main' (GitHub's standard)
PS> git branch -M main
5
Link to Your GitHub Repository & Push

Connect your local repository to GitHub and push your first commit:

PowerShell
# Add GitHub as the remote origin
PS> git remote add origin https://github.com/MatthijsKoen/My-AI-Voyage.git

# Verify the remote was added correctly
PS> git remote -v
# Expected:
# origin  https://github.com/MatthijsKoen/My-AI-Voyage.git (fetch)
# origin  https://github.com/MatthijsKoen/My-AI-Voyage.git (push)

# Push your first commit to GitHub
PS> git push -u origin main

# Windows will prompt for credentials:
# Username: MatthijsKoen
# Password: [paste your Personal Access Token here]
After the first push with -u origin main, future pushes only need git push ‚Äî the -u flag sets the default upstream once and for all.

Open the Extensions panel with Ctrl+Shift+X and install these:

Must Have

Visualize code authorship, compare branches, and see the full history of every line. The single most useful Git extension.

Must Have

Browse and edit your GitHub repos directly inside VS Code without cloning. Handy for quick edits.

Recommended

See your HTML blog update in real-time inside VS Code as you type ‚Äî no browser refresh needed.

Recommended

Recommended

Nice to Have

If you write blog drafts in Markdown before converting to HTML, this gives you full preview and shortcuts.

3
Configure VS Code Settings for This Project

Press Ctrl+Shift+P to open the Command Palette, type Open User Settings (JSON) and add these recommended settings:

VS Code ‚Äî settings.json additions
{
  // Auto-save whenever you switch away from a file
  "files.autoSave": "onFocusChange",

  // Format HTML automatically on save
  "editor.formatOnSave": true,

  // Show Git changes in the file explorer sidebar
  "scm.alwaysShowRepositories": true,

  // Enable Git autofetch to stay updated with GitHub
  "git.autofetch": true,

  // Confirm before syncing (prevents accidental pushes)
  "git.confirmSync": false,

  // Always show the Git status bar
  "git.enableStatusBarSync": true,

  // Word wrap for comfortable editing of long HTML lines
  "editor.wordWrap": "on"
}

For a fully visual workflow inside VS Code:

VS Code ‚Äî Publish Workflow
1. Edit your index.html in the editor
2. Press Ctrl+Shift+G to open Source Control panel
3. Under "Changes", click the + (Stage All Changes) button
4. Type your commit message in the box at the top
5. Press Ctrl+Enter to commit
6. Click the ‚Üë Sync Changes button in the status bar
7. Done! GitHub Pages will update in ~2 minutes ‚úì
"Sync Changes" in VS Code does a git pull followed by git push in one click ‚Äî it's the safest way to keep in sync with GitHub.

Writing Good Commit Messages

Your commit history is your blog's changelog. Use short, descriptive messages that explain what changed and why:

Good vs. Bad Commit Messages
# ‚ùå Bad ‚Äî tells you nothing useful

git commit -m "changes"

# ‚úÖ Good ‚Äî clear and descriptive

Quick Reference Cheat Sheet

The 10 Git commands you'll use 90% of the time.

Command What It Does
git status Show which files have changed since the last commit
git add . Stage ALL changed files, ready to commit
git add filename.html Stage one specific file only
git commit -m "message" Save a snapshot of staged files with a description
git push Upload your commits to GitHub (publishes the site)
git pull Download any changes from GitHub to your local copy
git log --oneline See a compact list of all past commits
git diff Show exactly what lines changed in unstaged files
git restore filename.html Discard changes to a file ‚Äî undo back to last commit
git log --oneline -10 Show last 10 commits in compact format
